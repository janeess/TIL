## Session 08 - 프레임워크  
개발시에 필요한 것들을 미리 만들어 두어서 작업을 편하게 해주는 코드 덩어리  
###  NestJS  
- 아키텍처와 구조를 잡아주는 프레임워크
- 데코레이터를 사용하여 추가적인 설정파일을 두지 않고 라우터나 미들웨어를 설정 가능
#### 장단점
- 장점 : 내부적으로 익스프레스와 패스트파이를 사용하고 있기 때문에 기존의 익스프레스의 미들웨어를 사용할 수 있음  
- 단점 : 자바의 스프링에 비해서는 기업용 프로그램을 작성하기에 부족한 부분들이 있음  
-> 최근 익스프레스의 발전이 더디기 때문에 NestJS가 앞으로 조금 더 발전가능성이 있다고 생각함  
###  Spring  
- 기업용 애플리케이션을 만드는데 필요한 것들을 모두 갖추고 있는 프레임워크 
- 탄생 자체가 기업명 애플리케이션 작성을 염두한 프레임워크이기 때문에 다른 프레임워크에 비해 많은 학습 필요
###  FastAPI
- 파이썬 커뮤니티에서 최근 인기있는 프레임워크
- API만 작성하는 용도라면 장고와 플라스크보단 FastAPI사용하는 것 추천
- 최소한의 설정과 제약을 주는 프레임워크, 플라스크와 비슷한 컨셉의 프레임워크 


## Session 09 - 데이터베이스  
검색과 축적이 쉽도록 정리된 데이터의 모음  
데이터 저장시 수 많은 문제가 생길 수 있는 상황에서도 데이터를 최대한 안전하게 보관,검색,수정,삭제할 수 있게 해주는 소프트웨어  
### 1. RDB (관계형 데이터 베이스)  
- 데이터를 행과 열로 이루어진 테이블로 관리하며 기본키를 사용해서 각 행을 식별
- 각 테이블 간의 관계를 지울 수 있음  
ex) 오라클, MySQL 등
### 2. NoSQL
- Not Only SQL : SQL뿐만 아니라 다른 것을 더 사용
- RDB는 데이터의 수정 삭제에 용이하다는 장점이 있지만, 성능이 낮음 -> 분산서버를 생각해야 함
- 그에 비해 NoSQL은 분산서버 관련한 부분들이 잘 되어 있음   
ex) MongoDB, redis  
#### MongoDB  
- 도큐먼트를 저장하는 디비
- 데이터로 JSON과 매우 비슷하게 생긴 BSON이라는 문서 모델 저장
- 클러스터를 지원해서 확장이 용이, 속도 빠름  
#### redis 
- 키배류 기반의 NoSQL 데이터베이스(키를 기반으로 데이터를 잘 관리할 수 있도록 해줌)
- 다양한 데이터 타입 지원&빠른 성능
